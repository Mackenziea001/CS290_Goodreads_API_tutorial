Goodreads' API Methods

The Good and the Bad

Goodreads is one of the most extensive user-populated database of books, but it's not perfect. Goodreads' moderator phrases it succinctly : "If you want a fun api to use that has great support, excellent documentation, client wrappers written in lots of different languages, and a variety of output formats, goodreads' api is not the one to use. If you want an API that has the best available book reviews, book metadata, and social book features on the internet, well, then you have no choice, you must use our api."

A Few Quirks

You can find the basics to all of the Goodreads API methods on their API page . This is an excellent resource, and it is mostly correct. If you're following documentation and something seems wrong, it might be that there's an error in what was coppied down or something has yet to be updated. The Goodreads Developers group is a great place to look for answers. For the most part, however, the official documentation is correct.

There are a few quirks to the set of methods listed, specifically the mixed use of JSON and XML, the mix of HTTP request methods in API methods, and the use OAuth.

Goodreads API methods return JSON and XML, but not always both.Most methods return just XML, a few return either JSON or XML depending on your preference, and just one, which allows you to get review statistics for a given list of ISBNs, returns JSON only. Essentially, if you'd prefer to not use JSON, no problem. Unfortunately, there's no getting around dealing with XML if you want to use the Goodreads API. Users have been asking the Goodreads support for more JSON endpoints for the past few years, but it doesn't look like that will happen any time soon. Both return types will be shown in the following examples.

Most of the methods use the GET request parameters, but some of those using OAuth use POST. Unfortunately, the Goodreads API is not completely standardized.

How to Use OAuth

While most Goodreads API methods do not use OAuth, it can be very useful, allowing users to login and access their own Goodreads data. From your webpage or app, users will be redirected to a page on Goodreads where they'll login and then be redirected back to your site or app via the callback URL. You'll be able to access their data using a token. There's some useful information here and you can see some an example of OAuth use with Python and Ruby at . This brief introductory guide to Goodreads' API doesn't cover the use of Oath, but there already exist many examples of how to use it here and even a library specifically designed to work around it's use here . However, as for all things, you can find more information about OAuth elsewhere online, as it will not be covered in this guide.
